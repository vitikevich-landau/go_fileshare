# 04. TUI-клиент «fileshare-commander»

## 1. Выбор библиотеки

| Вариант | Плюсы | Минусы |
|---|---|---|
| **FTXUI** ✅ | C++20, MIT, FetchContent (как остальные зависимости проекта), Windows+Linux, готовые компоненты (layout, фокус, события клавиатуры), активная разработка | моложе ncurses; свои представления о цикле событий |
| ncurses / PDCurses | вездесущ, классика, MC на нём и написан | голый API, весь layout руками, на Windows нужен PDCurses (другая библиотека), C-стиль |
| notcurses | мощный | тяжёлая зависимость, Linux-центричен |

**Решение: FTXUI.** Клиент в проекте кроссплатформенный (Windows/MSVC + Linux/GCC) —
FTXUI сохраняет это без `#ifdef`-зоопарка. Интеграция: `FetchContent`, как
nlohmann/json и GoogleTest.

## 2. Экраны приложения

Конечный автомат экранов:

```
 ConnectScreen ──connect──► ConnectingScreen ──AUTH_OK──► CommanderScreen
      ▲                          │ fail                        │
      └──────── ошибка ◄─────────┘                             │ Ctrl+D / quit
      ▲                                                        ▼
      └────────────── разрыв соединения (с диалогом) ◄─── DisconnectedDialog
```

### 2.1. ConnectScreen — вход

```
┌──────────────── fileshare commander ────────────────┐
│                                                       │
│   Сервер:      [ vps.example.com     ] Порт: [5555]   │
│   Логин:       [ vit                 ]                │
│   Пароль:      [ ************        ]                │
│   [x] Запомнить подключение   [ ] Запомнить пароль    │
│                                                       │
│   Сохранённые: > vps.example.com (vit)                │
│                  home-docker (admin)                  │
│                                                       │
│         [ Подключиться ]      [ Выход ]               │
└───────────────────────────────────────────────────────┘
```

- Список сохранённых профилей: Enter — подключиться, поля заполняются из профиля.
- Пароль при вводе маскируется.
- Валидация до отправки (пустой хост, порт вне диапазона).

### 2.2. ConnectingScreen

Спиннер + стадии: `TCP…` → `handshake…` → `аутентификация…` → `загрузка каталога…`.
Ошибка на любой стадии — красным, с человекочитаемой причиной
(`AUTH_FAIL: неверный пароль`, `UNSUPPORTED_VERSION: обновите клиент`) и
возвратом на ConnectScreen. Отмена — Esc.

### 2.3. CommanderScreen — основной экран

```
┌─ /home/vit/downloads ────────────┐┌─ fs://vps/incoming ──────── [SYNC●] ─┐
│  Имя            Размер   Дата    ││  Имя              Размер   Дата      │
│ /..                              ││ /..                                   │
│ /music                <ВВЕРХ>    ││ /video                     12.07.26   │
│  report.pdf     1.2M   10.07.26  ││ *new-build.tar.gz 340M     15.07.26   │
│  notes.txt       4K    01.07.26  ││  readme.txt        2K      03.05.26   │
│                                  ││ ~big.iso           4.5G    11.06.26   │
│                                  ││                                       │
├──────────────────────────────────┤├───────────────────────────────────────┤
│ 3 файла, 1.2M                    ││ 4 объекта, 4.8G · new: 1              │
└──────────────────────────────────┘└───────────────────────────────────────┘
 [download 34% ██████░░░░░░ 1.5G/4.5G  12.4 MB/s  eta 04:12 ]
 vit@vps:/incoming ❯
 F1 Help  F5 Get  ^R Refresh  F9 Admin  TAB Switch  F10 Quit
```

Приглашение командной строки — `login@server:path ❯` (акцентные логин и `❯`,
путь основным цветом); F-бар — «неоновые кейкапы»: клавиша циан-жирным, действие
приглушённым.

Составные части (сверху вниз):
1. **Две панели.** Каждая — локальная ФС **или** удалённая (`fs://имя-профиля/путь`).
   По умолчанию: слева локальная (cwd), справа удалённая (корень share). Активная
   панель подсвечена рамкой/заголовком.
2. **Статус-строка панели**: количество объектов, суммарный размер, счётчик новых.
3. **Строка передач**: активная закачка с прогрессом, скоростью и ETA; если очередь
   не пуста — `(+2 в очереди)`. Enter по строке открывает панель очереди.
4. **Командная строка** с приглашением `login@host:path$`.
5. **F-бар** — назначение функциональных клавиш, меняется по контексту
   (админу видна `9Админ`).

`[SYNC●]` в заголовке удалённой панели — индикатор соединения:
зелёный ● — подключено, жёлтый ◐ — реконнект, красный ○ — разрыв.

## 3. Клавиатура

Ядро (соответствие MC, чтобы руки не переучивать):

| Клавиша | Действие |
|---|---|
| `Tab` | переключить активную панель |
| `↑/↓ PgUp/PgDn Home/End` | навигация по списку |
| `Enter` | войти в директорию / (на файле) диалог действий |
| `Insert` / `Space` | пометить файл (мультивыделение), `*` — инвертировать |
| `F1` | помощь (клавиши, статус соединения) |
| `F2` | меню панели: сортировка (имя/размер/дата), показать скрытые, обновить |
| `F3` | просмотр файла (локально; удалённый — скачать во временный + просмотр, с подтверждением при > 1 MiB) |
| `F4` | инфо о файле: размер, mtime, checksum (`CHECKSUM_REQUEST`, `~` пока считается) |
| `F5` | **копировать**: с удалённой панели — скачать в директорию соседней; выделенные — очередью. (Локально→удалённо: заготовка под upload M13, пока «не поддерживается») |
| `F9` | админ-панель (только role=admin, см. [05-admin.md](05-admin.md)) |
| `F10` / `quit` | выход (с подтверждением при активных передачах) |
| `Ctrl+R` | обновить активную панель |
| `Ctrl+O` | скрыть/показать панели (полноэкранный лог операций) |
| `Esc` | отмена диалога / отмена текущей закачки (с подтверждением) |

Командная строка (минимальный набор, парсер `cli.hpp` переиспользуется):

```
cd <path> · get <path> [as <local>] · queue · info <path> · refresh
connect <profile> · disconnect · help · quit
```

Небуквенные нажатия уходят панели, набор текста — командной строке (как в MC).

## 4. Цветовая схема и индикация

Тема — «неоновый грид»: сдержанный киберпанк на базе палитры Tokyo Night
(раскладка MC осталась, 16-цветную классику сменил truecolor). Цвета заданы
hex-значениями и автоматически деградируют до 256/16 цветов средствами
lipgloss/colorprofile, поэтому старые терминалы остаются совместимыми.
Единственный источник палитры и стилей — `internal/tui/style.go`.

| Роль | Цвет | Использование |
|---|---|---|
| Фон баров | `#16161e` | F-бар, футер админ-панели |
| Поверхность | `#24283b` | неактивная подпись панели, отметка выделения |
| Рамки | `#3b4261` | карточки, разделительные линии |
| Текст | `#c0caf5` | основной текст |
| Приглушённый | `#565f89` | вторичный текст, подсказки |
| Неон-циан | `#7dcfff` | главный акцент: активная вкладка/заголовок, курсор, клавиши |
| Синий | `#7aa2f7` | директории (жирный, `/` префикс) |
| Пурпур | `#bb9af7` | роль admin, градиент прогресс-бара |
| Розовый | `#f7768e` | ошибки, опасные действия, плашка разрыва |
| Янтарь | `#e0af68` | **новые файлы** (жирный + `*` слева, см. §5), события, помеченные |
| Мята | `#9ece6a` | успех операции |
| Бирюза | `#73daca` | `.part`-файлы (`~` слева), трафик, приглашение |

Правила индикации:
- **ошибка никогда не исчезает молча** — она остаётся в логе операций
  (Ctrl+O), даже если плашка ушла;
- разрыв соединения — розовая плашка поверх удалённой панели
  (`⚠ CONNECTION LOST — reconnecting…`);
- **курсор не мигает**: во всех полях ввода статичный неоновый блок
  (`cursor.CursorStatic`), `textinput.Blink` нигде не запускается —
  ничего не моргает внизу экрана.

## 5. «Новое» — как определяется

Два механизма дополняют друг друга:

1. **Снапшот последнего визита.** Клиент хранит per-профиль
   `last_seen: unix_time` (обновляется при отключении). Всё с
   `mtime > last_seen` подсвечивается как новое. Работает даже без подписки.
2. **Live-события.** По `EVENT_FS(created/modified)` в текущей директории панель
   обновляется сразу, запись подсвечивается и попадает в лог операций
   (`+ появился: /incoming/new-build.tar.gz (340M)`).

Снять подсветку: `Ctrl+N` — «пометить всё как просмотренное» (обновляет `last_seen`).

## 6. Хранение профилей и паролей

Файл `~/.config/fileshare/profiles.json` (Windows: `%APPDATA%\fileshare\`),
права `0600`, создаётся с проверкой:

```jsonc
{
  "profiles": [
    {
      "name": "vps", "host": "vps.example.com", "port": 5555,
      "login": "vit",
      "secret": "<см. ниже>",          // опционально, если «запомнить пароль»
      "last_seen": 1752537600,
      "downloads_dir": "~/Downloads/fileshare"
    }
  ]
}
```

Пароль (если пользователь выбрал «запомнить»): хранится **не** сам пароль, а
`password_hash` (производная, нужная для challenge–response, см.
[06-security.md](06-security.md)) — компрометация файла не раскрывает пароль
открытым текстом, но позволяет войти на этот сервер, поэтому:
- по умолчанию галка «запомнить пароль» **выключена**;
- в UI это честно написано: «пароль хранится локально в ~/.config/fileshare».
Интеграция с системными кейрингами (libsecret/DPAPI) — осознанно вне scope v2.0,
отмечена как улучшение.

## 7. Внутреннее устройство (см. также 01-architecture.md §4)

- `AppState` — чистое состояние UI (панели, курсоры, очередь передач, лог) без
  FTXUI-типов: юнит-тестируется как обычный класс (навигация, выделение,
  формирование очереди, обработка событий соединения).
- `IFs` — общий интерфейс `LocalFs`/`RemoteFs`; панель не различает их. `RemoteFs`
  кэширует листинги (инвалидация по `EVENT_FS` и Ctrl+R).
- `Connection` — поток приёма + очередь исходящих; хартбит PING каждые
  `idle_timeout/3`; авто-реконнект с экспоненциальной задержкой (1→2→4→8с, макс 30с)
  и повторной аутентификацией; после реконнекта панели перечитываются.
- `TransferQueue` — последовательное выполнение закачек, каждая с retry (докачка
  по offset при обрыве — уже умеет клиент v1).
- `--batch`-режим: старый однострочный сценарий
  `fileshare-commander --batch --host H --port P --login L --get PATH [--out F]`
  (пароль — из `FILESHARE_PASSWORD` или интерактивно) — для скриптов и CI.

## 8. Тестирование TUI

- Вся логика в `AppState` → обычные GoogleTest'ы без терминала.
- `RemoteFs`/`Connection` — против реального сервера в тестах (как интеграционные
  тесты v1), плюс fault-injection: обрыв в середине листинга, EVENT во время
  закачки, реконнект.
- Смоук-тест отрисовки: FTXUI умеет рендерить в строку без терминала —
  golden-тесты на ключевые экраны (подключение, панели, диалог ошибки).
